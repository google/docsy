{{/* Returns whether the partial's REGION should render on the current PAGE under the params.td.lean_render build mode. The mode is off (every region renders) unless its value (case-insensitive) is one of: remove keeps only enough instances of each repeated, auto-generated region for a link checker to reach every link once. csr same dropping as "remove", but the missing left-nav is restored client-side (see _partials/sidebar.html and assets/js/csr-nav.js), so the lean build stays functionally equivalent to a full one. Either way, the regions kept are: navbar, footer configured chrome -> each home page sidebar computed left-nav -> each locale's docs landing page The string value names the mode, leaving room for a future "tag" mode. Reading a param (not os.Getenv) means no consuming site needs a security.funcs.getenv change.
