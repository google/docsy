{{/* Returns whether the partial's REGION should render on the current PAGE. With CSR (client-side chrome rendering) enabled via td.csr_enable, each repeated, auto-generated region is kept on just enough pages for a link checker to reach every link once; the dropped instances are restored in the browser (see _partials/navbar.html, footer.html, sidebar.html and assets/js/csr-nav.js) so the lean build stays functionally equivalent to a full one. The regions kept are: navbar, footer configured chrome -> each home page sidebar computed left-nav -> each locale's docs landing page When CSR is off, every region renders. Reading a param (not os.Getenv) means no consuming site needs a security.funcs.getenv change. See the Client-side chrome rendering guide for what this is and how to enable it: docsy.dev/content/en/docs/content/csr.md ctx: dict "page" PAGE "region" REGION */ -}} {{ $p := .page -}} {{ $render := true -}} {{ if partial "csr-enabled.html" $p -}} {{ if eq .region "sidebar" -}} {{ $root := cond (and (ne $p.Params.toc_root true) (eq $p.Site.Home.Type "docs")) $p.Site.Home $p.FirstSection -}} {{ $render = and $root (eq $p.RelPermalink $root.RelPermalink) -}} {{ else -}} {{ $render = $p.IsHome -}} {{ end -}} {{ end -}} {{ return $render -}}
