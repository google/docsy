## <i class="fa-solid fa-compass px-1"></i> Navigation and UX improvements {#navigation-and-ux}

### Active TOC entry tracking {#active-toc-entry-tracking}

<span class="badge text-bg-success rounded-pill">NEW</span> Docsy 0.13.0
introduces _active TOC entry tracking_, the _most upvoted_ enhancement request
of 2025. This feature highlights the TOC heading entry of the section that is
currently in view.

The current implementation uses Bootstrap [ScrollSpy][]. For details, including
**ScrollSpy limitations**, see [Active TOC entry tracking with
ScrollSpy][].[^scrollspy-note]

> <i class="fa-solid fa-bug fa-lg text-warning px-1"></i> **Numbered headings**
> will cause ScrollSpy to throw an exception. For details, see [#2329].

[^scrollspy-note]:
    This first implementation of TOC-entry tracking uses Bootstrap ScrollSpy.
    This may change if we cannot resolve ScrollSpy limitations and bugs.

[#2329]: https://github.com/google/docsy/issues/2329
[Active TOC entry tracking with ScrollSpy]:
  /docs/adding-content/navigation/#toc-entry-tracking
[ScrollSpy]: https://getbootstrap.com/docs/5.3/components/scrollspy/

### Section sidebar root

<span class="badge text-bg-success rounded-pill">NEW</span> Docsy 0.13.0
introduces the `sidebar_root_for` configuration option, which allows you to
scope sidebar navigation to specific sections. This is particularly useful for
large sites where you want different navigation trees for different sections,
and [docs-only] sites that also have non-docs sections.

To enable this feature, add `sidebar_root_for` to your page front matter.
Supported parameter values are `children` and `self`. For details and examples,
see the [Section sidebar root][] documentation.

[docs-only]: /docs/adding-content/content/#alternative-site-structure
[Section sidebar root]: /docs/adding-content/navigation/#sidebar-root

### Language menu visibility

The **language menu** visibility has changed. It is now displayed as follows:

- **Navbar**: displayed for _all_ screen sizes
- **Sidebar**: now hidden by default. This is a "**breaking**" UX change. For
  details on how to restore the legacy sidebar behavior, see [Adding a language
  menu][].

[Adding a language menu]:
  /docs/adding-content/navigation/#adding-a-language-menu

## <i class="fa-solid fa-universal-access text-info pe-1"></i> Accessibility improvements {#accessibility}

- <i class="fa-solid fa-palette fa-lg text-info px-1"></i> **Color contrast**
  has been improved throughout the theme, and Docsy now falls back to Bootstrap
  defaults for typography and color. This ensures better accessibility
  compliance out of the box. For details, see [#2285].
- <i class="fa-solid fa-bolt fa-lg text-warning px-1"></i> **Dark mode** [Flash
  Of Unstyled Content][] (FOUC) has been fixed ([#2332]).

[#2285]: https://github.com/google/docsy/issues/2285
[#2332]: https://github.com/google/docsy/issues/2332
[Flash Of Unstyled Content]:
  https://en.wikipedia.org/wiki/Flash_of_unstyled_content

## <i class="fa-solid fa-code px-1"></i> Alert shortcode improvements {#alert-shortcode}

As of Docsy 0.13.0, the `alert` shortcode **more fully supports Markdown
content**. This means that for Markdown shortcode calls like
`{{%/* alert */%}}`, your alert can:

- Contain calls to other shortcodes, that is, **nested shortcode** calls.
- Share **link definitions** from or with other parts of the page.
- Be used in lists and other **indented contexts**.

For details and examples, including important formatting requirements, see
[alert]. For implementation details, see PR [#941].

## <i class="fa-solid fa-triangle-exclamation fa-lg text-warning pe-1"></i> Breaking and other notable changes {#breaking-changes}

- <span class="badge text-bg-warning rounded-pill">BREAKING</span> **Alert
  shortcode** content is now processed differently, so you may need to review
  and adjust alert-content indentation, etc. For details, see [alert].

  > Previously, the shortcode used Hugo's `markdownify` function conditionally
  > based on the including page's file extension. This conditional logic has
  > been removed. Make sure to invoke your alerts using Markdown shortcode call
  > syntax `{{%/* alert */%}}`.

- <span class="badge text-bg-warning rounded-pill">BREAKING</span>
  **[Language menu](#language-menu-visibility)**: The language menu is now
  always displayed in the navbar. This change is not reversible via
  configuration[^lang-menu-style-note]. Legacy sidebar language menu visibility
  can be restored as described [previously](#language-menu-visibility).

- **Better NPM support**: Projects using [Docsy via NPM] will no longer face
  optional and peer dependency issues ([#2115]).

- **Translations** (**i18n**): added Occitan locale ([#2173]) and refreshed
  Simplified Chinese ([#2313]) and Ukrainian ([#2331]) [translation files][].

[^lang-menu-style-note]:
    But you _can_ restore the legacy language menu visibility in the navbar
    using CSS.

[Docsy via NPM]:
  /docs/get-started/other-options/#option-3-docsy-as-an-npm-package
[translation files]: /docs/language/#internationalization-bundles

## <i class="fa-solid fa-rocket fa-lg pe-1"></i> Upgrading to 0.13.0 {#upgrading}

### Prerequisites

> <i class="fa-solid fa-triangle-exclamation fa-lg text-warning px-1"></i> **We
> recommend** that you [Upgrade to Docsy 0.12.0][] first because it contains
> significant breaking changes.

### Upgrade procedure and AI help {#upgrading-and-ai}

> <i class="fa-solid fa-robot fa-lg text-info px-1"></i> Haven you used AI to
> help you upgrade Docsy? It can be a big help!

The [0.12.0 upgrade guide][] was written with both human project maintainers and
AI assistants in mind. In fact, I've successfully used it to upgrade projects
like [The Update Framework](https://theupdateframework.io) website. That is,
using only the upgrade guide as input, AI created [TUF PR #126] fully
autonomously. All I had to do was review it.

[0.12.0 upgrade guide]: /blog/2025/0.12.0/
[TUF PR #126]:
  https://github.com/theupdateframework/theupdateframework.io/pull/126

To upgrade from [0.12.0] to [0.13.0], follow the same steps as described in
[Upgrade to Docsy 0.12.0 from 0.11.0][upgrade-0.12], but use version 0.13.0
instead:

- **Docsy**: [0.12.0] → [0.13.0]<br> Includes **Bootstrap**: 5.3.6 → 5.3.8
- **Hugo**: 0.147.5 → 0.152.2&nbsp;[^vers-note]
- **Node**: LTS 22 → LTS 24&nbsp;[^vers-note]

[^vers-note]:
    These are the officially supported Node.js and Hugo versions associated with
    the named Docsy versions. Later versions may work, but are not officially
    supported.

After upgrading, review the [breaking changes](#breaking-changes) section below
and test your site thoroughly. For a testing checklist, see the [Upgrade to
Docsy 0.12.0][upgrade-0.12] guide.
