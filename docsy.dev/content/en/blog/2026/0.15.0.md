## Release Summary

Docsy 0.15.0 focuses on documentation-first site structures and experimental
agent-friendly content output.

- **[Doc-rooted sites](#doc-rooted-sites)**: Docsy now documents a supported
  pattern for publishing documentation pages at the site root, without a
  `/docs/` path prefix.
- **[Experimental agent-friendly output](#agent-friendly)**: Docsy adds
  experimental Markdown alternate outputs, a page-meta "View Markdown" link, and
  a generated `llms.txt` index.
- **[Community and footer links](#community-footer-links)**: links resolve more
  correctly with custom permalinks and multilingual sites, and footer links now
  support `rel`.
- **[`card` shortcode rendering](#card-shortcode)**: selected `card` shortcode
  fields now render in the page context.
- **Other updates**: deployment docs are split into focused pages, branch-model
  docs are clearer, and i18n coverage expands.

## {{% _param NEW %}} Doc-rooted sites {#doc-rooted-sites}

Docsy 0.15.0 improves support for documentation-first sites. A **doc-rooted**
site keeps page files in the `docs` section, but publishes those pages at the
site root instead of under `/docs/`.

This is useful when your site is primarily documentation and you want shorter
URLs such as `/get-started/` instead of `/docs/get-started/`.

See [Doc-rooted sites][] for the configuration pattern and the [Doc-rooted
example][] variant.

### Actions {#doc-rooted-sites-actions}

{{% _param CLEANUP %}} **Applies if** your project uses an older docs-only
configuration based on front matter `cascade` or `type` changes.

- Remove the older docs-only cascade configuration.
- Use Hugo `permalinks` configuration for the `docs` section.
- Check for path conflicts with `hugo --printPathWarnings`.

## {{% _param BADGE EXPERIMENTAL info %}} Agent-friendly output {#agent-friendly}

Docsy 0.15.0 adds experimental support for agent-friendly generated content:

- Markdown alternate outputs for home, section, and page content.
- A "View Markdown" page-meta link when a Markdown alternate exists.
- A generated top-level `llms.txt` index.
- Golden tests for Markdown output on `docsy.dev`.

This feature is opt-in for sites. The first implementation prefers a simple,
stable Markdown representation over reproducing the full visual page chrome.

### Actions {#agent-friendly-actions}

{{% _param BADGE EXPERIMENTAL info %}} **Applies if** you want to try
agent-friendly output.

- Enable Markdown outputs only for page kinds where the output is useful.
- Review generated Markdown for pages that depend heavily on shortcodes.
- Treat `llms.txt` as an experimental discovery aid and verify its links before
  publishing it on your production site.

## {{% _param BREAKING %}} Community and footer links {#community-footer-links}

Docsy 0.15.0 fixes community and footer link behavior:

- Footer links support `rel` attributes ([#2576][]).
- Community and footer links only open in a new target for external links
  ([#2576][]).
- Site-local community and footer links resolve under custom permalinks
  ([#2580][]).

This includes a potentially breaking change for multilingual sites: link paths
are now interpreted as site-relative paths.

### Actions {#community-footer-links-actions}

{{% _param BREAKING %}} **Applies if** your multilingual site configures
community or footer links with site-local paths.

- Review `params.links.user` and `params.links.developer` path values.
- If a link should target the default language, prefix the path with the default
  language code.
- Recheck generated community and footer links in each language.

For the configuration shape, see [Adding a community page][].

## {{% _param BREAKING %}} Card shortcode rendering {#card-shortcode}

Docsy 0.15.0 changes how the [`card` shortcode][] renders its `header`, `title`,
`subtitle`, and `footer` fields. These fields now use `$.Page.RenderString`
instead of `markdownify`, so they render in the page context ([#2565][]).

For most sites, this should make rendering more correct. It may affect projects
that override the shortcode or depend on the previous rendering behavior.

### Actions {#card-shortcode-actions}

{{% _param BREAKING %}} **Applies if** your project uses `card` shortcode
parameters with Markdown, images, links, or custom shortcode overrides.

- Review rendered cards that use `header`, `title`, `subtitle`, or `footer`.
- Check any local `layouts/_shortcodes/card.html` override against the upstream
  template.

## Other notable changes {#other-notable-changes}

### Internationalization

Docsy 0.15.0 includes i18n updates:

- Romanian locale and "View Markdown" translation ([#2583][], [#2603][]).
- Azerbaijan language YAML ([#2082][], [#2604][]).
- Missing German alert-label translations ([#2591][]).
- Baseline translations for the new "View Markdown" label ([#2602][]).

### Documentation and release process

Docsy 0.15.0 also includes documentation and project-process updates:

- Deployment docs are split into provider-specific pages ([#2556][]).
- The branch model documents `main`, `release`, `deploy/prod`, and `doc-rooted`
  ([#2572][]).
- Link linting now discourages full `docsy.dev` URLs for site-local links
  ([#2584][]).

## {{% _param FAS rocket primary %}} Upgrade to 0.15.0 {#upgrade}

Some upgrade steps are the same for each Docsy release. Follow the general
module-update flow from the previous release report, using version **0.15.0**
where that guide references 0.14.x.

For this release, review:

- **Docsy**: 0.14.3 → 0.15.0.
- **Hugo**: confirm the final supported Hugo version before publishing. The
  current `docsy.dev` tooling uses `hugo-extended` 0.157.0.
- **Node.js**: confirm the final supported Node.js range before publishing.

### Sanity checks

- Build your site locally with the final supported Hugo version.
- Check multilingual community and footer links.
- Check pages that use the `card` shortcode.
- If you enable Markdown outputs or `llms.txt`, inspect the generated `.md`
  pages and `/llms.txt`.
- If you use a doc-rooted configuration, run Hugo with `--printPathWarnings`.

## What's next?

The agent-friendly output work remains experimental. Follow [#2596][] for
continued work on generated Markdown, `llms.txt`, and related docs tooling.

For doc-rooted site follow-up, see [#2504][].
